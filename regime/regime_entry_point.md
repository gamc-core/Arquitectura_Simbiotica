# REGIME — ENTRY POINT 01

## PURPOSE

This document defines the minimal entry condition required for an actor to execute a structural experiment within the REGIME protocol.

Execution within REGIME is defined by structural acts and persistent records.

No prior membership, institutional affiliation or centralized authorization is required.

---

## MINIMAL ENTRY CONDITION

An actor may enter REGIME when the following sequence occurs:

condition  
→ act  
→ record  
→ evidence  
→ trajectory  

If the sequence exists, the experiment exists.

---

## ENTRY SEQUENCE

### 1. DECLARE CONDITION

A structural condition must be declared.

Examples:

- Finite operational window  
- Runtime activation  
- Artifact release  
- Node deployment  
- Experimental regime declaration  

---

### 2. EXECUTE ACT

A structural act must occur.

Examples:

- emit structural cell  
- release artifact  
- deploy runtime  
- execute infrastructure  
- publish execution trace  

---

### 3. PRODUCE RECORD

The act must generate a persistent record.

Examples:

- artifact repository  
- runtime output  
- structural cell  
- execution trace  
- persistent system record  

---

### 4. REGISTER EVIDENCE

Evidence must exist showing the act occurred.

Examples:

- repository commit  
- runtime log  
- artifact reference  
- persistent record identifier  

---

### 5. INTEGRATE INTO TRAJECTORY

The experiment becomes part of the accumulating structural history of the system.

---

## RULE

no act → no experiment  
no record → no evidence  
no trajectory → no system
